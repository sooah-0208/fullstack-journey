httpd, nginx, iis(window에 기본 설치되어있음/www서비스가 iis)  
        ↑  
url   http  
 ↓

          ↑  
conn   user,PWS  
 ↓  
STORAGE  (개인port)

- 각각의 프로그램은 포트가 필요함
- 포트넘버로 어떤 프로그램인지 구별 가능
- env로 포트 관리하는게 편함(접속 정보가 바뀌기때문)


## 동기 vs 비동기

- 동기(Sync): 하나 끝날 때까지 다음 못 함
- 비동기(Async): 기다리지 말고 다음부터 함

비동기를 사용하는 경우
- 서버 요청 (API)
- DB 조회
- 파일 읽기
- 타이머
- 네트워크 통신

❗ 이런 건 느리니까
동기로 하면 화면 멈추고, 서버 멈춘 것처럼 보임

## front와 back은 필수로 분리해줘야함!
그래야 back에서 문제가 발생해도 무한 로딩되지 않고 프론트에 문제발생 화면만 표출해 줄 수 있음

## api가 오류날 때
작업관리자모드에서 python강제로 껐다가 켜주기

## config
설정, 구성, 환경 설정 같은 뜻
예:
- DB 접속 정보 (아이디, 비번, 호스트)
- 서버 포트 번호
- API 키, 비밀 토큰
이런 애들을 configs파일로 묶어 관리하면 편함
