# Protocol
`규약`, 즉 컴퓨터나 원거리 통신 장비간 데이터 교환 통신 규칙 및 체계  
- 구성요소
    - 구문(Syntax): 데이터의 포맷(주고받는 데이터의 형식과 구조 정의규칙이라고 보면 됨),
                    신호 레벨(전압 높낮이 등 전기신호 기준),
                    부호화 방식(전송 가능한 신호 변환규칙 like ASCII, UTF-8)  
    ex) HTTP 프로토콜: Method(GET, POST) + URL + Version 이런 구조(웹 브라우저 <- 문서전송 -> 서버)   
        JSON, XML: Key:value로 쓸지 <key>value</key>로 쓸지 결정  
                   
    - 의미(Semantic): 특정 데이터 패턴의 해석 및 동작(오류 제어)
    - 타이밍: 전송 속도 및 순서 제어
- 종류
    - HTTP/HTTPS: 웹브라우저 서버 간의 문서 전송
    - TPC/IP: 인터넷의 기본 프로토콜, TCP는 전송 안정성, IP는 데이터 패킷의 주소 지정 및 라우팅 담당  
        *TCP vs IP*  
        1. IP는 상대의 연결여부를 확인하지 않고 일단 보냄 vs TPC는 상대가 연결되어 있어야만 전송함
        2. IP는 패킷이 유실되거나 순서가 바뀌어도 신경쓰지 않음 vs IP의 상위단계인 TPC는 패킷 유실시 재요청, 순서 섞이면 번호 재조립함.  
         => IP: 목적지 주소 찾기 -> TCP 안전한 박스에 담아 배달 -> HTTP 박스 열어 웹페이지로 사용자에게 제공
    - UDP: TCP와 같은 레벨인데 성능은 IP 같음 => 안전성은 떨어지나 빠름. 전송 속도가 중요한 실시간 방송이나 게임등에 유리
    - FTP: 보안성 없는 파일 전송 기능, 다중작업, 보안, 접속 정보확인 등에 사용(TCP, IP기반). 백업/ 원격서버관리 등에 사용됨 -> 보안 문제로 요즘엔 거의 안 씀
    - SSH: 원격 PC에 접속할 때 사용, 암호화 추가로 인증이 강화됨
    -SFTP: SSH방식으로 파일 전송
- 참고>> Semantic Kernel: 다양한 AI 모델과 도구들을 유기적으로 결합하여 AI 오케스트레이션을 구현하는 오픈소스 SDK

## AI Agent에서의 Protocol

`인공지능 agent <-> 인공지능 agent` & `AI agent <-> 시스템` 사이의 통신 표준 설정   
어떤 걸 명시하나요?  
=> 메세지의 문법, 구조, 순서, 에이전트의 대화 내의 역할, 메세지에 반응하는 방식 등을 명시  

### 이게 왜 필요하나요??  
**"서로 다른 두뇌(LLM)와 손발(Tool)을 가진 에이전트들이 공동의 목표를 위해 대화하는 표준 약속"**  

에이전트 기반 AI 시스템은 한 조직 내에서 부서별로 가진 데이터나, 타 조직이 가진 데이터는 각각 다른 방식으로 저장되어있음. 이를 `정보 사일로`라고 함.
- 정보 사일로란: 데이터나 시스템이 서로 공유되지 않고 고립된 상태
예) 스페인인, 일본인, 한국인간 소통을 할 때 스페인<->일본, 스페인<->한국, 일본<->한국, 각각을 통역해주는 통역사가 필요.
언어가 늘어날수록 통역사도 같이 필요해지게 됨. -> 이를 방지하고자 나온것이 공통어: `프로토콜`

- 프로토콜 사용의 장점(핵심 기능)
    1) 상호운용성이 좋아진다.
    - 에이전트가 자신 기본 구현 능력치와 상관없이 서로 소통이 가능해짐. -> 에이전트간의 협업 가능
    2) 개발 복잡성이 감소한다.
    - 상호작용시 복잡성을 추상화시켜주어 다중 에이전트 시스템을 구축할 때 과정을 간소화시켜줄 수 있음.
    3) 표준화로 통합을 원활하게 만들어준다.
    - 정해진 구조의 통신 방식을 제공하기때문에 기업간 호환성을 높여준다.

- 역할
1) task 위임: 상위 에이전트가 하위 에이전트에게 목표 할당 -> 복잡한 업무시 각자 할당량이 정해저 체계적인 업무가 가능해짐 + 병렬처리로 속도가 빨라짐
2) 상태 동기화: 각 에이전트가 어디까지 작업하였는지 실시간으로 공유 가능해짐
3) 자원 조율: 여러 에이전트가 충돌 없이 돌아가돌고 순서 정립

### 현시점에서 더 중요해진 이유
예전엔 GPT같은 거대 모델이 모든 task를 담당  
현 시점에서는 소형 에이전트들을 팀을 이루는 멀티에이전트 시스템이 대세로 떠오름  
각 에이전트가 업무 분담을 하여 전문화시킴 -> 업무 능력 향상  

또한 단순 생성을 너머 'Action'이 이루어짐  
단순 글을 생성하는 수순을 너머 직접 결제, 예약, 코드 실행등의 액션이 이루어지고있음  
-> 다른 시스템과의 통신이 절실해짐  

AI 생태계의 급격한 성장
Open AI, Google, Anthropic, 다양한 오픈소스 진영에서 각 프레임워크를 창출중임.  
단순히 '똑똑한 모델을 만들자'에서 '다양한 모델을 프로토콜로 엮어 가치 창출을 하자.'로 바뀌어가고있음.
